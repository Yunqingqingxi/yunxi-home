# 教训：chat_178

## 发生了什么
AI 被要求"把最新的 yunxi-home 从 GitHub 拉下来"，但：
1. 花了 9 轮（25+ 次工具调用）才向用户询问缺少的信息
2. AI 忘记了之前会话中向用户要过 GitHub Access Token
3. AI 尝试 `cd /opt/yunxi-home && git pull` 但该路径在本地不存在

## 根因
1. 没有跨会话记忆——AI 不知道 `/opt/yunxi-home` 是远程路径
2. 没有记住之前索要过的 Token 信息
3. 卡住检测未能触发——AI 每轮换不同工具，看起来"在做事情"

## 正确做法
1. **操作外部资源前先验证连通性**（git、SSH、scp 等）
2. **最多尝试 2 次**，无果后直接向用户提问，不要继续盲目尝试
3. **区分本地和远程操作**——不是所有路径都在本地
4. **检查认证状态**——`gh auth status` 比盲目 `git pull` 更有效

## Why
防止同类问题复发。此会话暴露了跨会话上下文丢失和卡住检测的根本缺陷。

## How to apply
- 每次涉及 git 操作、SSH 或外部资源访问时参考此记忆
- 严格遵守"尝试 2 次无果即询问"的原则
- 操作前先检查：目标是本地还是远程？有凭证吗？
